## 남은 작업 (우선순위 순, 이번 대화에서 논의됨)

1. **소경직/대경직/죽음 상태** — `OnGroggyFlinch`/`OnGroggyDown` C++ 이벤트는 이미 있음, 몽타주 반응 연결 필요. 전용 애니메이션 애셋 존재 여부 확인 필요. **죽음 처리 없으면 "몬스터 사망=게임 클리어" 흐름 자체가 끝나는 지점이 없어서 우선순위 높음.**
2. **패리 시 몬스터 움찔 모션** — 플레이어가 패리 성공하면 몬스터 현재 몽타주 중단 + 소경직보다 짧은 움찔 몽타주 재생. 1번(소경직)과 "현재 몽타주 중단하고 리액션 재생" 인프라를 공유하므로 1번 이후 진행 추천. 플레이어 쪽 기존 패리 시스템(`NJWs/ANS_ParryWindow`, `GE_ParryDamageBuff`) 확인 필요.
3. **뒷판정/포지셔닝 각도 임계값 튜닝** — 새 기능이라기보다 기존 30도/120도 임계값 조정에 가까움, 테스트하면서 계속 조절
4. **몬스터 공격에 나이아가라 이펙트** — 순수 연출, 급하지 않음
5. **2페이즈(체력 50% 이하 전환 + 원거리 패턴)** — 디자인 문서 TBD 항목 다수(브레스/원거리 세부사항 미정), 스코프 제일 큼. 1페이즈 루프(소경직~죽음)가 확실히 끝난 다음에 진행
6. **원거리/근거리 공용 돌진(Charge) 패턴 1~2회 랜덤 추가** — 이미 몽타주는 확보됨(꼬리치기도), 거리/각도 조건부 패턴 풀 확장 작업 때 같이
7. (설계 문서 TBD, 목록에 없었지만 아직 미구현) **전투 이탈**(NavMeshBoundsVolume 벗어나면 정찰 복귀) — 지금은 전투 진입하면 나올 방법이 없음
